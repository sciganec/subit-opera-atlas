# Opera Dynamics
## Динамічна модель оперної дії у просторі SUBIT

**Opera Dynamics** описує, як оперний твір рухається у структурному просторі WHO × WHERE × WHEN.  
Опера — це не статичний стан, а **послідовність переходів**, які формують драматургічну напругу, емоційну енергію та структурну логіку.

Динаміка опери визначається трьома рівнями:

1. **Суб’єктна динаміка (WHO → WHO)**  
2. **Просторова динаміка (WHERE → WHERE)**  
3. **Сезонна динаміка (WHEN → WHEN)**  

Разом вони утворюють **операційну траєкторію**.

---

# 1. Суб’єктна динаміка (WHO → WHO)

Це зміна джерела дії.  
Вона визначає, хто керує процесом і як змінюється суб’єктна перспектива.

## Основні переходи

### ME → YOU
Внутрішня інтенція переходить у взаємодію.  
Найпоширеніша драматургічна дуга.

### YOU → WE
Приватне стає колективним.  
Конфлікт розширюється.

### WE → THEY
Колектив стикається із зовнішньою силою.  
З’являється системний рівень.

### ME → THEY
Індивід проти структури.  
Один із найпотужніших векторів трагедії.

---

# 2. Просторова динаміка (WHERE → WHERE)

Це зміна енергетичного напрямку.  
Вона визначає, як розгортається дія у просторовому вимірі.

## Основні переходи

### SOUTH → WEST
Афект → хаос  
Емоційність переходить у нестабільність.

### WEST → NORTH
Хаос → структура  
Експеримент набуває форми.

### SOUTH → NORTH
Пристрасть → катарсис  
Класична трагедійна дуга.

### EAST → WEST
Порядок → інтуїція  
Рідкісний, але сильний вектор.

---

# 3. Сезонна динаміка (WHEN → WHEN)

Це найстабільніша динаміка — вона формує драматургічний цикл.

## Основні переходи

### SPRING → SUMMER
Ініціація → інтенсивність

### SUMMER → AUTUMN
Кульмінація → трансформація

### AUTUMN → WINTER
Зміна → завершення

### WINTER → SPRING
Завершення → новий початок

---

# 4. Комбіновані динаміки

Опера рідко рухається по одній осі.  
Типові комбіновані траєкторії:

## 4.1. ME / EAST / SPRING → YOU / SOUTH / SUMMER
Ініціація → конфлікт  
Класична любовна драма.

## 4.2. YOU / SOUTH / SUMMER → THEY / NORTH / WINTER
Конфлікт → система  
Історичні трагедії.

## 4.3. ME / WEST / AUTUMN → THEY / NORTH / WINTER
Криза → структура  
Модернізм і авангард.

---

# 5. Динамічні архетипи

Кожна опера має **динамічний профіль**, який складається з:

- **початкового стану**  
- **вектора розвитку**  
- **точки перелому**  
- **кінцевого стану**  

Це дозволяє описати будь-яку оперу як:

```
(WHO₁ / WHERE₁ / WHEN₁) → … → (WHOₙ / WHEREₙ / WHENₙ)
```

---

# 6. Точки напруги

Динаміка опери створює **зони максимальної напруги**:

## 6.1. Суб’єктні точки
- ME → THEY  
- YOU → THEY  

## 6.2. Просторові точки
- SOUTH → NORTH  
- WEST → NORTH  

## 6.3. Сезонні точки
- SUMMER → WINTER  
(різкий перехід без AUTUMN)

Ці точки визначають драматургічну силу твору.

---

# 7. Динамічні моделі

## 7.1. Трагедійна модель
```
SOUTH → WEST → NORTH
```

## 7.2. Психологічна модель
```
ME / WEST / SPRING → ME / WEST / AUTUMN → ME / NORTH / WINTER
```

## 7.3. Політична модель
```
YOU / SOUTH → WE / NORTH → THEY / NORTH
```

## 7.4. Авангардна модель
```
WEST → SOUTH → WEST
```

---

# 8. Динамічні порожнечі

Рідкісні або слабкі динаміки:

- EAST → EAST  
- YOU → EAST  
- WE / EAST / SPRING → WE / EAST / SUMMER  

Причина: низька драматургічна напруга.

---

# 9. Висновок

Opera Dynamics показує, що опера — це **рух**, а не стан.  
Драматургія виникає з переходів між:

- суб’єктністю (WHO)  
- напрямком (WHERE)  
- фазою (WHEN)  

Це дозволяє:

- аналізувати опери як динамічні системи  
- будувати енергетичні карти  
- створювати структурні інтерфейси  
- формувати SUBIT‑Opera‑64  

Opera Dynamics — це фундаментальний шар оперної аналітики.
